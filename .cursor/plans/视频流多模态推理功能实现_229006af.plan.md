# 视频流多模态推理功能实现计划

## 架构设计

### 前端流程

1. 用户点击摄像头按钮 → 打开摄像头权限
2. 实时捕获视频帧（30fps）
3. 每 10 帧抽取一次关键帧（约 3fps）
4. 关键帧编码为 Base64 JPEG
5. 发送到后端：`{frame: base64, text: "用户输入"}`
6. 接收 SSE 流式结果：文字 + 音频
7. 实时显示文字，合成语音播放

### 后端流程

1. 接收视频帧 + 文本
2. 解码 Base64 图像
3. 构建多模态消息：`[{"type": "image", "image": base64}, {"type": "text", "text": "..."}]`
4. 调用 qwen-3.5-flash 流式推理
5. 同时调用 TTS 合成语音
6. 通过 SSE 推送文字块 + 音频块

## 关键技术点

- **视频捕获**：Canvas + getUserMedia API
- **帧抽取**：requestAnimationFrame + 计数器
- **图像编码**：Canvas.toBlob() → Base64
- **多模态消息**：LangChain ImageContent + TextContent
- **流式推理**：LangChain streaming + SSE
- **并发合成**：TTS 与 LLM 推理并行
